# Valid Parentheses

## Problem Statement
- given a string $S$ containing just the characters `'('`, `')'`, `'{'`, `'}'`, `'['` and `']'`, mathematically prove if the input string is valid.
- an input string is formally defined as valid if and only if:
- open brackets must be closed by the exact same type of brackets.
- open brackets must be closed in the correct topological order (LIFO).
- every close bracket has a corresponding open bracket of the same type.


## Approach: Push-down Automaton (LIFO Stack)

- this problem maps directly to a fundamental theoretical concept: parsing a language generated by a Context-Free Grammar. This requires a Push-down Automaton (PDA), which we implement practically using a [[Stack]].

- **initialize:** Create an empty stack.
- **iterate:** For each character $c$ in the string $S$:
   - if $c$ is an **opening bracket** (`(`, `{`, `[`), push its expected **closing bracket** (or push $c$ itself) onto the stack. Pushing the expected closing bracket simplifies future equality checks.
   - if $c$ is a **closing bracket** (`)`, `}`, `]`):
     - check if the stack is empty. If it is, we have a closing bracket with no mathematical precedent (unmatched). Return `false`.
     - check the top of the stack. If it does not equal $c$, the topological order is violated (mismatched bracket). Return `false`.
     - otherwise, the brackets match. Pop the top element from the stack.
- **termination:** After processing all characters, the stack must be completely empty. If elements remain, it implies there were unclosed open brackets. Return `stack.empty()`.

> [!tip]
> A micro-optimization involves verifying the parity of $|S|$. If the string length is odd, it is mathematically impossible for all parentheses to be paired. We can immediately return `false`.


## Code Implementation

```cpp
#include <string>
#include <stack>
using namespace std;

bool isValid(const string& s) {
    // Parity check optimization
    if (s.length() % 2 != 0) return false;
    
    stack<char> st;
    
    for (char c : s) {
        // Push the expected inverse to simplify matching logic
        if (c == '(') {
            st.push(')');
        } else if (c == '{') {
            st.push('}');
        } else if (c == '[') {
            st.push(']');
        } else {
            // Unmatched closing bracket or topological violation
            if (st.empty() || st.top() != c) {
                return false;
            }
            st.pop();
        }
    }
    
    // Valid if and only if all opened scopes are successfully collapsed
    return st.empty();
}
```


## Complexity Analysis
- **time Complexity:** $O(N)$ where $N = |S|$. We traverse the string linearly. Stack operations `push()` and `pop()` execute in strict amortized $O(1)$ time.
- **space Complexity:** $O(N)$. In the worst-case scenario, the string consists entirely of opening brackets (e.g., `"(((((((("`), causing all $N$ characters to be pushed onto the auxiliary stack memory.
